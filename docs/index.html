Claude Code とは

Anthropic が出している、ターミナルで動く AIコーディングエージェント。

  • 普段のシェルで claude と打って起動するだけ
  • ファイル編集・bash 実行・web 検索・git 操作 を 自分で判断して 連続実行する
  • 「カーソルで補完」ではなく、「タスクを丸投げ」 するための道具
  • 料金は Claude のサブスク (Pro / Max) に含まれる従量。月固定で使い放題感覚で叩ける

具体的に何をやってくれるか (例)

  • 「このリポジトリを読んで、〇〇のバグを直して PR 出して」
  • 「app.py に new endpoint を足して、テスト書いて、CI 通るか確認して」
  • 「nikkei.com の昨日のトップ記事 10 本取ってきて、要約して Notion 化」
  • 「このパイプラインで AIデータセンターのショート1本作って、YouTube にアップして」

← まさに今日見せるやつ。1〜2 行の指示で 30 秒の動画 + アップ完了。

"AIエディタ" との一番の違い

従来 (Cursor / Copilot)

  • カーソルにいる行を 補完する
  • 1 ファイル単位の操作が中心
  • エディタを開いてないと使えない

Claude Code

  • タスク全体を 分解して計画→実行
  • 複数ファイル / シェル / API を 横断
  • エディタ不要。CI でも cron でも回せる

Coding Agent が "得意" な領域

逆に言うと、これ以外はまだ人間がやるほうが早い。

得意

  • 大量・反復的なコード変更 (リネーム・移行)
  • 複数ファイル横断のリファクタリング
  • テスト書き / バグの再現と修正
  • 使ったことない言語 / FW を試す
  • 定型スクリプト・CLI ツール作成
  • ログ・データ分析の "とりあえず可視化"
  • README / API ドキュメント生成
  • 外部API調査 + クライアント実装

まだ苦手

  • 仕様が曖昧な要件定義 (人が決める)
  • ビジネス判断 (KGI/KPI の選択)
  • 顧客との対話 / 政治
  • 感性が要るデザイン判断
  • 長期 (数週間以上) のステート管理
  • セキュリティクリティカルな最終承認

職種別 "持って帰れる使い方"

職種使いどころ
エンジニア 多ファイル横断リファクタ / テスト書き / 既存コードベース理解 / オープンソース contribute / DSL 自作
IT コンサル クライアントデータの一次分析 (csv→可視化) / 業界調査ノートの構造化 / 提案資料の下書き / RFP の要件抽出

他のツールとの比較

ツール形態強み弱み
Claude Code CLI エージェント 長時間・多ファイル耐性、Skill / MCP で拡張、サブスクで定額 エディタ統合は控えめ (IDE プラグインはある)
Cursor IDE (VSCode fork) エディタ統合が滑らか、デザイナーフレンドリー 1ファイル中心、長尺タスクは苦手
GitHub Copilot IDE プラグイン シェアNo.1、入力補完が高速 "エージェント" 感は弱い (最近の Agent モードはまだ未成熟)
OpenCode / Aider CLI (OSS) 自前API キーで動く、安価 長時間タスクの再開・状態管理は弱い
ChatGPT / Claude (Web) チャットUI 気軽。設計の壁打ちには最強 ファイル編集・bash 実行は手動コピペ

僕は Cursor + Claude Code 併用。コード補完は Cursor、自動タスクは Claude Code。

ニュース系のときの追加スタック

  • HeyGen Photo Avatar — 冒頭 5 秒のアナウンサーを実写で生成して挿入
  • news_fetch.py — Nikkei / Bloomberg / CoinPost を Playwright で取得 (paywall 含む)
  • x_trending.py — X API v2 で @nikkei や @BloombergJapan の高エンゲ投稿を抽出
  • news_image_ref.py — Bing 画像検索の結果を nanobanana に参照画像として渡し、著作権セーフな「描き直し」を行う

画像生成: 何を使うか

バックエンドコスト品質速度判定
Gemini 2.5 Flash Image (nanobanana) $0.039/枚
Batch で $0.020		 ◎ 安定 ◎ 速い 採用
OpenAI gpt-image-1 (ChatGPT) $0.04 - $0.17 ○ △ 代替可
FLUX.2 ローカル (Mac MLX) 無料 △ ◎ 却下
Stable Diffusion ローカル 無料 △ △ 却下

FLUX.2 は 2026-05 に試して却下。無料は魅力だが QA に時間がかかり、トータルコストで nanobanana のほうが安かった。

無料の選択肢を本気で探したけど…

  • FLUX.2 (MLX) — 速い (Mシリーズ Mac) けど、人物の整合性 / 細部が崩れがち。QA で人間が直す時間が膨らんで、結局 nanobanana のほうが安い
  • LTX-2.3 / Wan2.2 (i2v) — 静止画→動画も試したが品質不足。29GB 消費して削除
  • Stable Diffusion — チューニングしないとアニメ系は厳しい。LoRA / Checkpoint 管理コストが高い
  • 結論 — 個人用途 (月数百〜数千枚) なら、無料を追わずに nanobanana / gpt-image-1 で月 $20-50 を払う ほうが時間的にも金銭的にも得

日本語 Aivis は VOICEVOX 互換 API。自前で hub.aivis-project.com から音声モデル落として叩く。 複合語の誤読 (「起業家 → きぎょういえ」) があるが user_dict_word で辞書登録すれば直る。

1日の運用ループ

  1. 朝: youtube_stats_sync で前日までの views を DB へ同期
  2. Redash の "Top 30 動画" を眺めて、伸びてるトピック・チャンネルを把握
  3. X / Nikkei / Bloomberg を x_trending.py で取得 → 今日のネタを 1〜2 個選ぶ
  4. Claude Code に「このネタで偉人伝ショート作って」と一行投げる
  5. シナリオ確認 → OK → Claude が生成・アップまで自動
  6. 夕方: 同じく ニュース系を 1〜2 本

実稼働 30 分 / 日で 5〜6 本投下、を 1 ヶ月続けて累計 339k 再生まで来た。

拡張: Skill と MCP

  • Skill — /news-shorts のように / で呼び出せる、Markdown で書かれた "Claude への指示書"。 ドメイン知識・好み・運用ルールを Skill にすると、Claude がその場で読み込んで挙動を変える。
  • MCP (Model Context Protocol) — Claude に "外部ツール" を生やすプロトコル。 Notion / Slack / Linear / 自社DB などを Claude から直接叩けるようになる。

ハンズオン

各自、自分のマシンで 1 本作ってみる。30 分。

  1. リポジトリを clone: git clone https://github.com/<owner>/webtoon-gen
  2. .env.example をコピーして .env を作り、GEMINI_API_KEY を入れる
  3. Aivis Speech をローカル起動 (port 10101)
  4. uv sync で依存インストール
  5. claude 起動
  6. 「動物雑学チャンネル用に、ハチドリの面白いショートを作って」と投げる
  7. シナリオ確認 → OK → 出力された mp4 を open で確認

うまく行かないとき

  • Gemini が "no image returned" — プロンプトが規約に触れた可能性 (実在人物の顔, ロゴ, 暴力)。プロンプトを抽象化
  • Aivis が誤読 — tools/aivis_userdict.py に登録 + seg_*.mp3 と narration.mp3 を削除して再生成
  • 音声直したのに動画に反映されない — output/.../narration.mp3 と narration_with_sfx.mp3 も concat キャッシュなので削除必須
  • YouTube quota 超過 — 1 日 30 本まで、15:00 JST にリセット。Playwright (tools/yt_pw_upload.py) で迂回可
