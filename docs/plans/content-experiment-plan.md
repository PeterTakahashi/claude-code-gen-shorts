# コンテンツ実験計画 — 初期ユーザー獲得のための検証動画制作

作成日: 2026-05-14

## 背景・目的

将来的に「動画生成エージェント（desktop アプリ、AI 従量課金、web プラットフォーム連携）」を商品化する前段階として、現状のパイプライン（静止画 + ナレーション + 章末クリフハンガー型 webtoon 動画）で**ストーリー性に強い動画**を作り、SNS でアピールして初期ユーザーを獲得する。

商品の差別化軸：
- プロンプトから一気に派手なアニメ動画を作る競合と違い、**静止画 + 音声 + 強いシナリオ**で勝負する
- 重要なのはアニメーションではなく**ストーリー**である、というプロダクト哲学
- 静止画ベースゆえに**多言語展開の限界コストが極めて低い**（後述）

検証の目的：
1. SNS で「AI エージェントでこういう動画が作れる」とアピールし初期ユーザーを呼ぶ
2. ジャンル別に視聴維持率・反応を測り、商品ターゲット層を見極める
3. 現状のパイプライン（特にシナリオ生成）の弱点をジャンル別に洗い出す
4. **多言語展開の ROI** を実測する

## 検証スコープ

- **3 ジャンル** を並行で実施
- 各ジャンル **2 動画**（最低）〜**3 動画**（最大）
- 各動画 **10〜30 分の長尺**
- 国際展開対象動画は **JP + EN を同時リリース**（追加言語は反応を見て）
- YouTube 長尺をマスター、TikTok / Instagram / X に短尺切り抜きを派生
- 各動画の説明欄に「AI エージェントが生成しました」のアピール文 + 将来の web プラットフォームへの導線を入れる

## 多言語・国外展開戦略

本プロダクトの構造的優位は、**コストの大半を占める画像生成が言語非依存**である点にある。画像内テキストは原則禁止（`feedback_no_japanese_in_images` で既ルール化）なので、同じ panel セットがそのまま全言語で再利用できる。シナリオは LLM が言語ごとに書き分け、音声は TTS で言語ごとに生成する — どちらも画像生成に比べれば桁違いに安い。

### コスト構造の意味するところ
- 画像生成: 1 動画あたりの最大コスト、**全言語で共有（再利用率 100%）**
- シナリオ LLM 翻訳・ローカライズ: 1 言語あたり数 USD レベル
- 音声生成: Aivis (JP, ローカル無料) / ElevenLabs 等 (EN・その他) — 言語ごとに再生成、コスト低
- 結論: **1 動画の追加言語あたりの増分コストはマスター制作コストの数 %**。日本語版を出すなら英語版を出さない理由がほぼない

### 初期ターゲット言語（優先順）
1. **日本語**（マスター、Aivis、`feedback_subtitle_conventions` に準拠）
2. **英語**（最重要、グローバル YouTube 市場最大）
3. 反応を見て **スペイン語 / ポルトガル語 / 中国語繁体字 / 韓国語** を追加（YouTube 視聴者上位言語）

### 言語別のチャンネル・アカウント戦略
- **YouTube**: 言語別チャンネル運用が基本（Baltoon JP / Baltoon EN）。多言語音声トラック機能も併用検討
- **TikTok / Instagram**: 必ず言語別アカウント（アルゴリズムが地域別最適化されるため共用は損）
- **X**: 言語別アカウント

### 多言語展開で検証したい仮説
- 画像コストが言語間で完全に償却できるため、多言語化の ROI は本当にプロダクトの構造的優位となるか
- ジャンル別に **EN 市場での反応が JP より良いもの**はあるか（例: チャレンジャー号は EN で爆発するか）
- 多言語 SNS アカウントの運用負荷（コメント対応、CS）は許容範囲か
- 自動字幕でなく**ネイティブ TTS ナレーション + ローカライズ済シナリオ**が、海外視聴者の維持率を上げるか

## ジャンル 1: 実話犯罪・未解決事件（昭和の事件簿系）

### 候補題材
- 三億円事件
- グリコ森永事件（かい人 21 面相）
- 世田谷一家殺害事件
- 八つ墓村のモデル事件（津山 30 人殺し）
- Zodiac Killer
- D.B. Cooper

### 第 1 弾推奨: 三億円事件
- 日本人視聴者の認知度が高い
- 写真資料が少なく webtoon 化（漫画調再現）の価値が出る
- 時系列で謎が深まる構造（白バイ警官 → モンタージュ → 時効）はクリフハンガー多発

### 第 2 弾推奨: Zodiac Killer
- 英語版を併走させればグローバル展開可能
- 暗号・手紙という強い視覚要素
- 未解決ゆえに「視聴者と一緒に推理する」体験が成立

### このジャンルの強み・差別化
- 既存競合「ゆっくり解説」との差別化: 合成音声でなく自然なナレーション + 漫画調ビジュアル
- 「上品な実話系」ポジションが空いている
- TikTok/YouTube で「実話系」は既に巨大ジャンル → 検索流入が見込める

### このジャンルで検証したい仮説
- webtoon の panel 表現が、写真ベースのドキュメントよりも没入感を出せるか
- 章末クリフハンガーで離脱率を抑えられるか（K-ドラマ式）

### 第 1 弾推奨: Theranos

### 第 2 弾推奨: ライブドア
- 日本人視聴者向け、堀江貴文の認知度
- 「ITベンチャー vs 検察」のドラマ性
- 2000 年代の時代背景を webtoon で描き直す意義

### このジャンルの強み・差別化
- 章構造が題材に内蔵（創業 → 急成長 → 綻び → 崩壊）→ シナリオ生成 LLM の負荷が下がる
- ビジネス層は YouTube 長尺の視聴単価（CPM）が高い
- 図表・株価チャートを panel に組み込めば情報密度が上がる
- 将来のツールユーザー層（起業家・経営者）と視聴者層が重なる

### このジャンルで検証したい仮説
- ビジネス層の長尺視聴維持率が他ジャンルより高いか
- 数字・図表系の panel を pipeline で自動生成できるか（現状の弱点）

## 実行スケジュール（提案）

| Phase | 期間 | 内容 |
|-------|------|------|
| Phase 1 | 2〜4 週 | 各ジャンル第 1 弾を制作（計 3 本） |
| Phase 2 | 1 週 | SNS 同時アップロード、初動測定 |
| Phase 3 | 2〜4 週 | 反応を見て各ジャンル第 2 弾を制作（計 3 本） |
| Phase 4 | 継続 | 短尺切り抜き・リミックスを SNS に投下し露出継続 |

## SNS 別の配信戦略

- **YouTube**: 長尺フル動画をマスター配信。SEO 狙い（題材名 + 「真相」「ドキュメント」「完全版」）
- **TikTok**: 60 秒の核心抜粋、章末クリフハンガーで「続きは YouTube」誘導
- **Instagram Reels**: webtoon の縦スクロール特性を活かした panel 連続演出
- **X**: スレッド形式で panel を貼り、最後に YouTube リンクで誘導

## KPI（計測指標）

- ジャンル別の視聴維持率（特に章末でドロップしないか）
- 投稿あたりのチャンネル登録者増加数
- 動画 → 将来のツール認知（説明欄リンク CTR）
- ジャンルごとのコメント数・質（「これ AI で作ったの？」系のリアクション）

## パイプライン側で改善が必要そうな点（仮説）

検証を通じてどれが実際にボトルネックになるかを切り分ける。

- **シナリオ生成の章末クリフハンガー強化** — ジャンル別テンプレートが必要
- **実在人物の顔の一貫性** — `[[feedback_image_consistency]]` で既知
- **図表・年表・地図 panel の自動生成** — 企業・災害ジャンルで必須、現状未対応
- **シリアスなドキュメント調ナレーション** — Aivis で表現可能か検証
- **多言語ナレーション基盤** — 英語（ElevenLabs 等）以降の TTS 選定・声の一貫性・発音辞書
- **シナリオの言語別ローカライズ** — 単純翻訳でなく、文化的注釈の差し替え（例: 三億円事件で円建て金額を出すか、米国向けには 1968 年当時の USD 換算を入れるか）
- **言語別字幕の出し分け** — `feedback_subtitle_conventions` は JP 用ルール、EN/他言語用の表記規則を別途定める必要

## 残課題・要決定事項

- **怪談・都市伝説ジャンルを Phase 2 で追加するか？**（前回議論した八尺様・コトリバコ等は SNS 拡散力が高いが、将来ビジネス層に売る際のブランド毀損リスクあり）
- **検証期間中の月予算（API 費用、特に画像生成）** — 画像コストは多言語で償却できるため、英語版を出す前提で予算を立てるべき
- **YouTube 公開チャネル構成**: Baltoon JP / Baltoon EN を分けるか、多音声トラックで 1 チャンネルに統合するか
- **EN 版の TTS ベンダー**: ElevenLabs / OpenAI / Azure のどれを使うか、声の一貫性をどう担保するか
- **第 1 弾 3 本（三億円事件 / Theranos / JAL123 便）の制作順序と、各動画で同時公開する言語**
